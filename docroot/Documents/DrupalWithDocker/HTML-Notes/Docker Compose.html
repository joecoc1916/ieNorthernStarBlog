Run Sets of Containers

Part of Docker's fundamental design was the idea that one container only runs one process.


In this tutorial, we'll:

  • Introduce Docker Compose
  • Run multiple containers at once using Docker Compose
  • Identify the purpose of docker-compose.yml
  • Learn what resources a set of containers share


Goal

Understand the purpose of Docker Compose and install it to run sets of containers.



Docker Compose

This grouping is sometimes called a "stack" or a "container set".

docker-compose.yml

The Compose file includes containers' names, network connectivity, restart policy, and many, many more details.

Once the .yml file is created, use the docker-compose command to manage the containers described in docker-compose.yml.  Docker-compose sets up a virtual network for us between the various containers, easch with there own node.  Compose also runs a Domain Name Server (DNS) inside the network for us.

Each of these containers needs to talk to each other in order to run our Drupal sites.  Compose helps us here as well.

  2. Define the services that make up your app in docker-compose.yml so they can be run together in an isolated environment.

  3. Run docker-compose up and Compose starts and runs your entire app.

Furthermore, it is best to always place the docker-compose.yml file under version control.

That way, any changes to the file can be tracked, and all developers can share the same configuration of containers for the project. This cuts down on it-works-on-my-machine problems by standardizing the project's development infrastructure.
