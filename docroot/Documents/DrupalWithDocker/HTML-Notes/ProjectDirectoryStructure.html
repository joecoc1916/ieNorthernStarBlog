Example 1: Project root outside of web-accessible docroot directory. (USING).

my-web-site-project
- .git
- .docker-compose.yml
- build-scripts/
- docroot/
 - index.php

The project directory name is significant!

Always place the docker-compose.yml file in the project root because of the way Compose auto-generates container names. This name is only unique to each system; two containers running on different machines will have different container IDs, but can have the same name.  Docker Compose relies on the name of the directory containing docker-compose.yml to name containers.

Line 1.  The version key must be at the beginning of the file,

The services section

version: '3'
services:
  web:
    image: osiolabs/drupaldevwithdocker-php
  db:
    image: osiolabs/drupaldevwithdocker-mysql

The above has 2 containers in it:

  • The web container uses an image that was created specifically for this series, and provides Apache and PHP 7.
  • The db container also uses an image that was created specifically for this series. It provides a MySQL server.

Fortunately, there are many, many container images we can use; many particularly useful to us as Drupal developers.
